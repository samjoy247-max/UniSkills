  • স্টুডেন্টরা নতুন স্কিল শিখতে চান এবং অন্যদের স্কিল অফার করতে পারেন
  • অ্যালামনাইরা ক্যারিয়ার পরামর্শ এবং মেন্টরশিপ প্রদান করেন
  • অ্যাডমিনরা সমস্ত কন্টেন্ট যাচাই এবং অনুমোদন করেন
💡 কেন Form Data (`application/x-www-form-urlencoded`) ব্যবহার করা হয়?
এটি একটি ঐতিহ্যবাহী Django ওয়েব অ্যাপ্লিকেশন যা JSON REST API নয়:
  • 📌 HTML ফর্ম ব্যবহার করে ডেটা পাঠায় (ব্রাউজার থেকে সরাসরি)
  • 📌 CSRF সুরক্ষা এর জন্য প্রয়োজন (প্রতিটি POST-এ `csrftoken`)
  • 📌 Session-ভিত্তিক প্রমাণীকরণ ব্যবহার করে (JSON টোকেন নয়)
  • 📌 HTML পৃষ্ঠা রেসপন্স হিসাবে ফেরত পাঠায়, JSON নয়

📝 Module 1: রেজিস্ট্রেশন এবং লগইন

স্টুডেন্ট রেজিস্ট্রেশন GET, POST 🔓 PUBLIC
GET, POST /accounts/register/student/
নতুন স্টুডেন্টরা এই এন্ডপয়েন্টে তাদের একাউন্ট তৈরি করে। ইমেইল অবশ্যই @uap-bd.edu দিয়ে শেষ হতে হবে।
🔍 বিস্তারিত ব্যাখ্যা:
এটি একটি দুই-ধাপ প্রক্রিয়া:
  • GET অনুরোধ: ফর্ম খালি অবস্থায় প্রদর্শন করে
  • POST অনুরোধ: ব্যবহারকারী ফর্ম পূরণ করে জমা দেয়
📮 কেন Form Data ব্যবহার করি?
আমরা HTML ফর্ম থেকে ডেটা পাই। ব্রাউজার স্বয়ংক্রিয়ভাবে `application/x-www-form-urlencoded` ফরম্যাটে পাঠায়। এটি URL-এনকোড করা নাম-মূল্য জোড়ার একটি সহজ, নিরাপদ উপায়।

📥 প্রয়োজনীয় ফর্ম ফিল্ডস (POST-এর জন্য)

ফিল্ড নাম ডেটা টাইপ বাধ্যতামূলক বর্ণনা
username Text (String) ✓ হাঁ অনন্য ইউজারনেম (অন্য কেউ ব্যবহার করতে পারবে না)
email Email ✓ হাঁ অবশ্যই @uap-bd.edu দিয়ে শেষ হতে হবে
department Text (String) ✓ হাঁ বিভাগ (উদাহরণ: CSE, BBA, EEE)
university_id Text (String) ✓ হাঁ আপনার স্টুডেন্ট আইডি নম্বর
password1 Password ✓ হাঁ শক্তিশালী পাসওয়ার্ড (কমপক্ষে 8 অক্ষর)
password2 Password ✓ হাঁ পাসওয়ার্ড নিশ্চিতকরণ (password1 এর সাথে একই হতে হবে)
csrftoken Token ✓ হাঁ Django CSRF সুরক্ষা টোকেন (ফর্মে স্বয়ংক্রিয়ভাবে অন্তর্ভুক্ত)

📤 সম্ভাব্য প্রতিক্রিয়া

✅ GET - সফল অনুরোধ (ফর্ম দেখানো)
200 OK
Content-Type: text/html
কী ঘটে: ব্রাউজার স্টুডেন্ট রেজিস্ট্রেশন ফর্ম দেখায়
✅ POST - সফল রেজিস্ট্রেশন
302 Found (Redirect)
কী ঘটে:
  • নতুন CustomUser তৈরি হয় (role='student')
  • is_alumni_verified = False সেট হয়
  • ডেটাবেসে সংরক্ষণ করা হয়
রিডাইরেক্ট করে: /accounts/login/ সাফল্যের বার্তা সহ
❌ POST - ব্যর্থ রেজিস্ট্রেশন
200 OK (Validation Error)
Content-Type: text/html
কী ঘটে: ফর্ম আবার দেখানো হয় ত্রুটি বার্তা সহ:
  • ❌ ইমেইল @uap-bd.edu দিয়ে শেষ না হলে
  • ❌ পাসওয়ার্ড মিল না খেলে (password1 ≠ password2)
  • ❌ ইউজারনেম ইতিমধ্যে থাকলে
  • ❌ কোনো ফিল্ড খালি থাকলে
অ্যালামনাই রেজিস্ট্রেশন GET, POST 🔓 PUBLIC
GET, POST /accounts/register/alumni/
অ্যালামনাইরা এই এন্ডপয়েন্টে নিজেদের অ্যাকাউন্ট তৈরি করে। তবে লগইন করার আগে অ্যাডমিন অনুমোদন প্রয়োজন।
🔑 গুরুত্বপূর্ণ পার্থক্য:
স্টুডেন্টদের থেকে আলাদা: অ্যালামনাই অ্যাকাউন্টে is_alumni_verified = False থেকে শুরু হয় এবং লগইন করতে পারে না যতক্ষণ না অ্যাডমিন এটি যাচাই করে।
📮 Form Data ফরম্যাট
অ্যালামনাই রেজিস্ট্রেশনও একটি HTML ফর্ম ব্যবহার করে যা `application/x-www-form-urlencoded` হিসাবে পাঠানো হয়।

📥 প্রয়োজনীয় ফর্ম ফিল্ডস

ফিল্ড নাম ডেটা টাইপ বাধ্যতামূলক বর্ণনা
username Text ✓ হাঁ অনন্য ইউজারনেম
email Email ✓ হাঁ যেকোনো বৈধ ইমেইল ঠিকানা
graduation_year Integer ✓ হাঁ কবে গ্র্যাজুয়েট হয়েছেন (বছর)
major Text ✓ হাঁ আপনার বিশেষত্ব/বিভাগ
current_company Text ✓ হাঁ বর্তমান কর্মস্থল
password1 Password ✓ হাঁ পাসওয়ার্ড
password2 Password ✓ হাঁ পাসওয়ার্ড নিশ্চিতকরণ
✅ POST - সফল রেজিস্ট্রেশন
302 Found (Redirect)
কী ঘটে:
  • নতুন অ্যালামনাই ব্যবহারকারী তৈরি হয়
  • is_alumni_verified = False সেট হয়
  • অ্যাডমিনদের অনুমোদনের জন্য অপেক্ষা করতে হবে
রিডাইরেক্ট করে: /accounts/login/ বার্তা: "অ্যাডমিনের অনুমোদনের অপেক্ষা করুন"
ব্যবহারকারী লগইন GET, POST 🔓 PUBLIC
GET, POST /accounts/login/
যাচাইকৃত ব্যবহারকারীরা তাদের ইউজারনেম এবং পাসওয়ার্ড দিয়ে লগইন করে। এটি একটি সেশন তৈরি করে যা পরে অন্যান্য সুরক্ষিত পেজে ব্যবহৃত হয়।
🔐 সেশন-ভিত্তিক প্রমাণীকরণ: লগইন করলে, ব্রাউজার একটি সেশন কুকি পায়। প্রতিটি পরবর্তী অনুরোধে এই কুকি পাঠানো হয় যাতে সার্ভার জানতে পারে কে ব্যবহারকারী।

📥 প্রয়োজনীয় ফর্ম ফিল্ডস

ফিল্ড নাম ডেটা টাইপ বাধ্যতামূলক বর্ণনা
username Text ✓ হাঁ ইউজারনেম অথবা ইমেইল ঠিকানা
password Password ✓ হাঁ পাসওয়ার্ড
csrftoken Token ✓ হাঁ CSRF সুরক্ষা টোকেন (স্বয়ংক্রিয়)
✅ GET - লগইন ফর্ম (সফল)
200 OK
Content-Type: text/html
কী ঘটে: খালি লগইন ফর্ম প্রদর্শিত হয়
✅ POST - সফল লগইন
302 Found (Redirect)
কী ঘটে:
  • ডাটাবেসে পাসওয়ার্ড যাচাই করা হয়
  • সেশন কুকি সেট করা হয়
  • ব্যবহারকারী সফলভাবে প্রমাণিত
রিডাইরেক্ট করে: তাদের ড্যাশবোর্ডে (রোল অনুযায়ী):
  • স্টুডেন্ট → /accounts/student_dashboard/
  • যাচাইকৃত অ্যালামনাই → /accounts/alumni_dashboard/
  • অ্যাডমিন → /admin/
❌ POST - ব্যর্থ লগইন
200 OK (Auth Error)
Content-Type: text/html
ত্রুটির কারণ:
  • ❌ ইউজারনেম বা পাসওয়ার্ড ভুল
  • ❌ অ্যালামনাই যাচাইকৃত নয় (is_alumni_verified=False)
  • ❌ ব্যবহারকারী বিদ্যমান নেই
প্রতিক্রিয়া: লগইন ফর্ম ত্রুটি বার্তা সহ পুনরায় প্রদর্শিত হয়
লগআউট GET 🔓 PUBLIC
GET /accounts/logout/
ব্যবহারকারীর সেশন শেষ করে এবং তাদের লগ আউট করে।
✅ GET - সফল লগআউট
302 Found (Redirect)
কী ঘটে:
  • সেশন কুকি মুছে ফেলা হয়
  • সার্ভারে সেশন ডেটা সাফ করা হয়
  • ব্যবহারকারী সম্পূর্ণভাবে লগ আউট
রিডাইরেক্ট করে: /accounts/ (ল্যান্ডিং পেজ)
