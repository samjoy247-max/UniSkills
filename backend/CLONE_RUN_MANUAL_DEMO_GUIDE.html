UniSkills Clone & Demo Guide

This page explains how to clone the repo on another PC, run the project without manual environment setup, log in with the default admin account, and present the features step by step to your sir.

1. Clone and run

Beginner-friendly setup

Step 1: Clone the repository

Open VS Code or a terminal and clone your GitHub repo.

git clone https://github.com/samjoy247-max/UniSkills.git
cd UniSkills

Step 2: Open the backend folder

Go to the backend project root.

cd "FINAL UX/backend"

Step 3: Create and activate a virtual environment

If the clone does not already have a working venv, create one.

python -m venv .venv
.venv\Scripts\activate

If your system uses another Python path, use that interpreter instead.

Step 4: Install dependencies

pip install -r requirements.txt

Step 5: Run migrations

This project now works even without a manual .env.

python manage.py migrate

After migration, the default admin account is created automatically.

Step 6: Start the server

python manage.py runserver 127.0.0.1:8000

Open http://127.0.0.1:8000/ in your browser.

2. Default accounts

Ready-to-use login

After migrate, this account exists automatically:

  • Username: admin
  • Password: admin
  • Admin panel: /admin/
Important: No extra .env file is needed for the clone-and-run path. The backend now falls back to SQLite and console email in development.

3. Manual demo script

How to show every feature to your sir

  1. Landing page: Open the home page and briefly tell that UniSkills has student, alumni, booking, rating, and admin features.
  2. Student registration: Go to /register/student/, use a UAP email, and submit the form.
  3. OTP verification: If console email is active, check the server terminal for the OTP. If SMTP is enabled later, check inbox/spam.
  4. Student login: Log in with the new student account and show the dashboard.
  5. Skill post creation: Create a skill post and explain it is pending until admin approval.
  6. Admin login: Log in with admin/admin at /admin/ or the admin login route.
  7. Moderation: Approve or reject the skill post and show the status update on the student side.
  8. Search and filter: Go to the skill browsing page and search/filter by keyword, category, or mode.
  9. Booking flow: Create a booking request, then open the provider side and accept/decline it.
  10. Rating flow: After a completed session, submit a rating and show the rating history/dashboard.
  11. Alumni flow: Register or open alumni pages, show profile and alumni post features.
  12. Final recap: Mention the app supports authentication, profile, moderation, browsing, booking, rating, and admin management.
4. Troubleshooting

If something breaks

Common bug warning: If a bug appears while you manually test, note the exact page, button, and browser message before changing code. That makes debugging much faster.
  • Server will not start: Make sure you are inside FINAL UX/backend and that the virtual environment is activated.
  • Port already in use: Stop the old Django server and run it again.
  • Database error on a fresh clone: The app now falls back to SQLite automatically when MySQL env vars are missing.
  • OTP not appearing in inbox: In default development mode it prints to the server console instead of email.
  • Admin login missing: Run python manage.py migrate again; the post-migrate hook creates admin/admin.
